Guidelines & handoff

How to use the system day-to-day — voice, accessibility, composition rules, and a cheatsheet for developers and Claude.

Voice & tone

VetVet operates in a high-trust, high-stakes space: veterinarians sell the practice of their careers here. We write like a trusted broker — clear, warm, never casual, never salesy.

✓ Do
“Your listing is under review. You’ll hear back within 2 business days.”
Plain, specific, sets a clear expectation.
✕ Don’t
“🎉 Awesome! Your listing is on its way to stardom!”
Emoji, hype, vague timelines — breaks trust.

Copy rules

  • Sentence case for all UI copy including buttons and headings.
  • No exclamation points in product UI. Warmth comes from precision, not punctuation.
  • Use dollars, not $ in marketing copy; use $1,250,000 in data contexts.
  • Say listing, not practice listing. Say buyer, not prospect.
  • Button verbs commit: Submit listing, Request access, Withdraw offer. Avoid “Click here.”

Composition rules

How to arrange the components so every screen feels like the same product.

  1. One primary action per screen. If you feel two are needed, step back — the screen probably has two jobs.
  2. Left-align by default. Center alignment reserved for empty states, marketing pages, and modal CTAs.
  3. Scan the page top-to-bottom in 3 seconds. If a user can’t identify the primary task from scale + color alone, rework hierarchy before adding copy.
  4. Cards are containers, not shadows. Use borders (neutral-border) first; reach for --shadow-default only when a card must float.
  5. Semantic color = state. Green means success. Amber means attention. Red means destructive. Never use them decoratively.

Primary vs. secondary action

✓ Do
One committed action; alternative action is quiet.
✕ Don’t
Two primary buttons dilute the CTA and confuse users.

Accessibility

We target WCAG 2.2 AA. Follow these and you’ll pass by default.

  1. Color contrast. All text on white is at minimum 4.5:1. subtext-color (#78716C) on white is 4.67:1 — use at 14 px+.
  2. Focus rings. Every interactive element uses the --shadow-focus token on focus-visible. Don’t remove it.
  3. Hit targets. Interactive elements are at least 32×32 px; 44×44 on touch surfaces.
  4. Never color alone. Status uses color + icon + text. Error fields get aria-invalid="true" + descriptive message.
  5. Motion. Animations under 250 ms; respect prefers-reduced-motion.

For developers

Wire the system into your app in four steps.

  1. Drop vetvet-ds.css into your static assets (or import from the package — see the Engineering wiki).
  2. Reference it once in your app shell (see snippet below).
  3. Use semantic classes (.btn, .input, .badge) or the CSS variables directly.
  4. Add your own product CSS on top — never override the DS from inside it. Ship a DS patch instead.
// 1. link the stylesheet (once)
<link rel="stylesheet" href="/ds/vetvet-ds.css"/>

// 2. use primitives
<button class="btn btn--brand-primary">Submit listing</button>

// 3. or reference tokens in your own CSS
.listing-card {
  background: var(--color-surface);
  border: 1px solid var(--color-neutral-border);
  border-radius: var(--radius-lg);
  padding: var(--space-6);
  box-shadow: var(--shadow-default);
}

For Claude & designers

The minimum context to give Claude when building a new VetVet screen. Copy-paste into your prompt.

// Design system: VetVet v1.0
// Aesthetic: calm, high-trust, precise. Forest teal + stone neutrals.
// Radius: small (4–8 px). No gradients. No emoji in product UI.
// Type: Inter. H1 24/600, H2 20/600, Body 14/400, Caption 12/400.

// Core semantic tokens (CSS vars in vetvet-ds.css):
--color-brand-primary     #14535A   // primary buttons, headlines
--color-brand-secondary   #299198   // interactive accents
--color-brand-tertiary    #E6F2F3   // brand tints, badge bg
--color-default-font      #1C1917
--color-subtext-color     #78716C
--color-neutral-border    #E7E5E4
--color-surface           #FFFFFF
--color-default-background #FBFBF9

// Layout rules
- 4 px spacing grid; use --space-* tokens
- One primary action per screen (.btn--brand-primary)
- Cards: 1px border, radius-lg, 24 px padding
- Tables: dense, hover row, neutral-50 header
- Shadows: sm for inputs, md for menus, overlay for modals

Governance

How the system evolves without breaking what ships.

Need something new? Open a ds-request issue with a screenshot and the use case. Triaged weekly.
One-off exception? Allowed inside your feature only, tagged with a // ds-exception comment linking the issue.
Breaking changes are shipped as major versions with a 2-sprint deprecation window. No silent rug-pulls.
Ownership. Design Systems team owns tokens + primitives. Product teams own compositions (listing card, deal room, etc.).
